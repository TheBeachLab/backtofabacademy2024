The file name encodes the week I‚Äôm in and the chapter. Chapter 00 is the header. I use chapters 10, 20, 30‚Ä¶ to develop the sections of the week. If a chapter gets too long, I subdivide it using intermediate numbers: 20, 21, 22, etc. Chapter 90 is always the conclusion.

AI-driven automatic translations

At Fab Academy, documentation has to be in English. Traditional language translators are pretty bad. They can‚Äôt understand context and produce results that don‚Äôt sound natural. I‚Äôll be using artificial intelligence to translate the text of the chapters into English and also into German.

The AI model must be able to recognize Markdown syntax and respect it. The model may improve (or even change) throughout Fab Academy. That‚Äôs why I‚Äôll keep the text in Spanish and rerun the translations on all files from time to time. I will only edit the files in Spanish. I won‚Äôt manually manipulate the generated translations. So, if what you read in English or German makes no sense, blame OpenAI or whichever model I‚Äôm using.

I asked C√©sar Garcia, from La Hora Maker, to help me find a model for translation. C√©sar recommended using OpenAI‚Äôs Whisper API, which can translate directly from Spanish audio. For now, I‚Äôm only interested in the translation, so I created an assistant in the OpenAI API with these instructions:

Translate the text from Spanish to English. Read the entire document to grasp the context before translating it. Take into account nuances and idioms of the Spanish language and translate them into their English equivalents. The translation should not be literal, focusing on maintaining the original meaning and providing a translation that makes sense in English. Only translate the text within quotes in the YAML header of the Markdown file. Do not translate links, URLs, blocks of code, and code snippets. If encountering a markdown link, only translate the text inside square brackets. Recognize and retain brands and names without translation. Use correct grammar and syntax in the final text. The style of the translation should be informal, with a touch of sarcastic humor.

I have another model in German with similar instructions. I keep changing the instructions from time to time to try and improve the translation.

This page you‚Äôre reading has about 4000 tokens. You can find out how many tokens a text has using the OpenAI Tokenizer. The cost of translating this page into the two languages is approximately 0.32 USD, considering that every 1000 tokens cost 0.01 USD for the input and 0.03 USD for the output. It seems pretty expensive to me, and the cost will go up as Fab Academy progresses. For that reason, I will only translate contents when I consider the work advanced.

For the moment, the English translations are quite good. I asked Sophia D√∂ring to also review the German translation. She said that, in general, it‚Äôs also quite good, although sometimes it uses German words that are infrequently used, especially for referring to technical terms.

Project Management

I‚Äôd like to dedicate a few lines to explain how I manage my projectsDon‚Äôt let anyone tell you how to manage a project. It‚Äôs something very personal and you have to create your own system.

. When you don‚Äôt have a boss, and nobody tells you what to do, you have to be very disciplined. Otherwise, you might fall into a negative spiral.

The most important thing is knowing where you‚Äôre going. Because if you don‚Äôt, you‚Äôve got a serious problem. Sometimes you‚Äôll find yourself lost in Fab Academy (and in your life). You don‚Äôt know what you want to do, you don‚Äôt feel like doing anything, and it seems like you‚Äôre moving in slow motion while the world races by. When that happens, remember Phil Stutz. Second is learning to reduce and simplify. One of your biggest problems in Fab Academy will be something like this: You can‚Äôt find a photo that you‚Äôre sure you had. You don‚Äôt know if it‚Äôs in your phone‚Äôs photos, on Google Drive, or on a USB drive, if you moved it to your computer‚Äôs folder, or if someone gave it to you on WhatsApp‚Ä¶ Simplify. Reduce.

And lastly, and I think most importantly, the real secret to achieving anything, no matter how difficult, is by taking many, very small steps. You solve a small problem, then another, and then another. That‚Äôs how it works.

As for tools to help you manage, I‚Äôve tried a few. So I‚Äôll start with the ones that don‚Äôt work for me:

  • Kanban-like software and similar. They try to mimic Post-it notes. I don‚Äôt use them because: I find them complicated and they don‚Äôt let me draw.
  • Microsoft Project and the like. No comment.
  • Web services like Notion.com, Monday.com, and similar. I‚Äôm not going to spend a second of my short life trying to understand how a service that only seems easy on TikTok (and will probably shut down tomorrow afternoon) works.

Now, I‚Äôm using a combination of plain text for long-term goals, reminders for medium-term goals, and Apple‚Äôs Freeform for the tasks I‚Äôm going to do today.
