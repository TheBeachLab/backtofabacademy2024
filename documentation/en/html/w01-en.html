Using Markdown

Here, I don‚Äôt want to be too innovative. The strategy that has worked very well for me for many years is to write content in a plain text file in a format called Markdown .md. This way, I focus only on writing the content. Advantages of writing in Markdown:

  • It‚Äôs easy to write; you don‚Äôt have to contort your fingers typing </h1> and stuff like that.
  • It‚Äôs easy to apply styles and organize the text.
  • It can be read without feeling like you‚Äôre decoding The Matrix.

File Structure

In this way, I am generating Markdown files with documentation in Spanish. At first, I wrote the text for each week in a single file. But as you‚Äôll read later on, I use a paid service for translation. Every time I edited a line, I had to translate the whole file again. To cut down on costs, I now divide the week into chapters and create a file for each chapter. This way, only the chapters I‚Äôve modified get translated. My file structure is inspired by the programming language BASIC:

/documentation
    /es
        /md
            /w01
                w01-chapter-00-es.md
                w01-chapter-10-es.md
                w01-chapter-20-es.md
                w01-chapter-21-es.md
                ...
                w01-chapter-90-es.md
                w01-chapter-99-es.md

The name of each week‚Äôs file encodes the week number, chapter, and language of the documentation. Chapter 00 is the header. I use chapters 10, 20, 30‚Ä¶ to develop the sections of the week. If a chapter is too long, I subdivide it using the intermediate numbers: 20, 21, 22, etc. Chapter 90 is always the conclusion, and 99 is the footer.

Later, I concatenate all the chapters of the week into a single file, in this case: w01-es.md.

Important Note: I never manually edit the concatenated file. I only edit the separate chapters.

AI-Powered Automatic Translations

At Fab Academy, all documentation must be in English. Traditional language translators often fall short. They fail to understand context and produce results that don‚Äôt sound natural. I plan to use artificial intelligence to translate the text of the chapters into English and also into German.

The AI model needs to be capable of recognizing Markdown syntax and respecting it. It will also need to handle internal links, because the links that go from w02-es.md in the Spanish documentation need to be changed to w02-en.md in the English version of the documentation. It‚Äôs possible that the model might improve (or even change) throughout fab academy. Therefore, I‚Äôll keep the text in Spanish and rerun the translation of all the files from time to time. I will only edit the files in Spanish. I won‚Äôt manually manipulate the generated translations. So, if what you read in English or German doesn‚Äôt make sense, blame OpenAI or whatever model I‚Äôm using.

I asked C√©sar Garcia, from La Hora Maker, to help me find a model for the translation. C√©sar recommended using OpenAI‚Äôs Whisper API, which is capable of translating directly from Spanish audio. For now, I‚Äôm only interested in translation, so I‚Äôve created an assistant in the OpenAI API with the following instructions:

I have another model in German with similar instructions. I‚Äôve been changing the instructions from time to time to try to improve the translation.

The page you‚Äôre reading is about 4000 tokens. You can find out how many tokens a text has using the OpenAI Tokenizer. Automating the Entire Process

To automate the entire process, I converted a Bash script I made for the educational program FabZero into Python. I write something like this:

python auto.py --translate updating week 1

The script translates the updated chapters if it finds --translate among the arguments. I do this to save costs. Then, it concatenates all chapters and creates a single Markdown file for each week. The next step is converting all these files into HTML. During the conversion, if it finds a link to a markdown document, it converts it into a link to its corresponding HTML document using this LUA filter. Finally, it uploads everything to Github as long as there is a message, which in this case is updating week 1. If there‚Äôs no message, it doesn‚Äôt carry out any of the processes related to git.

Project Management

I‚Äôd like to explain how I manage my projects. When you have no boss, and no one tells you what to do, you need to be very disciplined. Otherwise, you might fall into a negative spiral.

The most important thing is knowing where you‚Äôre headed. Because if you don‚Äôt, you have a very serious problem. Sometimes you‚Äôll find yourself lost in the Fab Academy (and in your life). You won‚Äôt know what you want to do, you won‚Äôt feel like doing anything, and it seems like you‚Äôre moving in slow motion while the world speeds past you. When that happens, remember Phil Stutz. Invest in yourself: Exercise, reconnect with other people, write your memories in a journal. It helps me, and I‚Äôm convinced it will help you too. Soon, you‚Äôll see your North Star. Head towards it.

The second thing is learning to reduce and simplify. One of your biggest problems in Fab Academy will be similar to this: You can‚Äôt find a photo you‚Äôre sure you had. You don‚Äôt know if it‚Äôs in your phone‚Äôs photos, on Google Drive, or on a USB drive, if you moved it to a folder on your computer, or if someone sent it to you on WhatsApp‚Ä¶ Simplify. Reduce.

And lastly, and I believe most importantly, the real secret to achieving anything, no matter how difficult it is, is to make many small steps. Solve a small problem, then another, and then another one. That‚Äôs how it works.

In terms of tools to help you manage, I‚Äôve tried a few. So I‚Äôll start with the ones that don‚Äôt work for me:

  • Post-it notes on the wall. The idea is good. Very visual and agile. But it has two problems. First problem: In Barcelona, it‚Äôs often very hot, and the notes fall off. Second problem: If I can‚Äôt see the wall, there‚Äôs no project.
  • Kanban-style software and similar. They try to mimic Post-it notes. I don‚Äôt use them because: They seem complicated and don‚Äôt let me draw.
  • Microsoft Project and similar. No comment.
  • Web services like Notion.com, Monday.com, and similar. I refuse to spend a single second of my short life trying to understand how a service that only looks easy on TikTok and will probably shut down tomorrow afternoon works.

Now, I‚Äôm using a combination of plain text for long-term goals, reminders for medium-term goals, and Apple‚Äôs Freeform for tasks I‚Äôm going to do today. Freeform is an app with an infinite canvas, similar to Miro. It has the advantages of Post-it notes without their disadvantages. I can customize it and create my own system.
